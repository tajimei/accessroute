# アクセシビリティ監査チェックリスト

AccessRoute アプリの各画面に対するアクセシビリティ監査チェック項目。
各リリース前に確認すること。

---

## 1. VoiceOver 動作確認

### 共通項目

- [ ] 全てのインタラクティブ要素に `accessibilityLabel` が設定されている
- [ ] `accessibilityHint` で操作結果が説明されている
- [ ] 画面遷移時に VoiceOver フォーカスが適切な要素に移動する
- [ ] モーダル表示時に背景要素が `accessibilityElementsHidden` で隠される
- [ ] ローディング中は `accessibilityValue` で状態が通知される
- [ ] エラーメッセージが `UIAccessibility.post(.announcement)` で読み上げられる

### ホーム画面（地図）

- [ ] 地図上のピンに場所名が `accessibilityLabel` として設定されている
- [ ] 現在地ボタンに「現在地に移動」等のラベルが設定されている
- [ ] 検索バーにフォーカス移動でき、プレースホルダーが読み上げられる
- [ ] 地図のズームイン/アウトボタンにラベルが設定されている
- [ ] ルート表示時、概要情報（距離・所要時間）が読み上げられる

### プロファイル設定画面

- [ ] 移動タイプ選択（車椅子、ベビーカー等）のセグメントコントロールが読み上げられる
- [ ] 各設定項目のオン/オフ状態が `accessibilityValue` で通知される
- [ ] スライダー（距離設定）の現在値と単位が読み上げられる
- [ ] 保存ボタンのラベルが設定されている

### チャット画面

- [ ] チャットメッセージの送信者（ユーザー/AI）が区別されて読み上げられる
- [ ] 新しいメッセージ到着時にアナウンスメントが発行される
- [ ] 入力フィールドにフォーカス移動でき、送信ボタンが操作できる
- [ ] AI 応答のローディング状態が通知される

### ルート結果画面

- [ ] ルート候補リストの各項目に距離・所要時間・スコアが読み上げられる
- [ ] アクセシビリティスコアの意味が説明される（例: 「アクセシビリティスコア85、100中」）
- [ ] 警告（階段あり等）が適切に読み上げられる
- [ ] ルート詳細のステップ一覧が順序付きで読み上げられる

### スポット詳細画面

- [ ] スポット名、カテゴリ、住所が読み上げられる
- [ ] アクセシビリティ情報（車椅子対応、エレベーター等）の状態が読み上げられる
- [ ] 写真に代替テキスト（`accessibilityLabel`）が設定されている
- [ ] 電話番号・ウェブサイトリンクが適切にラベル付けされている

---

## 2. Dynamic Type 確認

### 共通項目

- [ ] 全テキスト要素が `UIFont.preferredFont(forTextStyle:)` を使用している
- [ ] `adjustsFontForContentSizeCategory = true` が設定されている
- [ ] 最大サイズ（AX5）でもレイアウトが崩れない
- [ ] テキストの切り詰め（truncation）が発生する場合、全文が別途確認できる
- [ ] 固定高のセルを使用していない（自動レイアウト対応）

### サイズ別確認

- [ ] 最小サイズ（xSmall）: テキストが判読可能
- [ ] デフォルトサイズ（Large）: レイアウトが意図通り
- [ ] 最大アクセシビリティサイズ（AX5）: スクロールで全内容閲覧可能
- [ ] サイズ変更時にリアルタイムで反映される（`traitCollectionDidChange` 対応）

---

## 3. 高コントラスト・カラーブラインド対応

### コントラスト比

- [ ] 通常テキスト: 背景とのコントラスト比 4.5:1 以上（WCAG AA）
- [ ] 大テキスト（18pt以上）: コントラスト比 3:1 以上
- [ ] アイコン・インタラクティブ要素: コントラスト比 3:1 以上
- [ ] フォーカスインジケーター: 背景とのコントラスト比 3:1 以上

### 高コントラストモード

- [ ] `UIAccessibility.isInvertColorsEnabled` に応じた表示調整
- [ ] `accessibilityIgnoresInvertColors` が画像・地図に設定されている
- [ ] `UIAccessibility.isDarkerSystemColorsEnabled` で文字が太くなる

### カラーブラインド対応

- [ ] 色だけで情報を伝達していない（アイコン・テキストを併用）
- [ ] アクセシビリティスコアの高低が色以外（数値・アイコン）でも判別可能
- [ ] 警告表示が色以外の視覚的手がかり（アイコン、ボーダー）を持つ
- [ ] ルート表示の色分けにパターンや太さの違いを併用
- [ ] シミュレーターで以下のモードで確認済み:
  - [ ] 第一色覚（Protanopia）
  - [ ] 第二色覚（Deuteranopia）
  - [ ] 第三色覚（Tritanopia）

---

## 4. 最小タップターゲット

### 基準

- [ ] 全てのタップ可能な要素が 44x44pt 以上
- [ ] ボタン間の間隔が 8pt 以上

### 画面別確認

- [ ] ホーム画面: 地図上のピン、ナビゲーションボタン
- [ ] プロファイル画面: セグメントコントロール、トグルスイッチ
- [ ] チャット画面: 送信ボタン、入力フィールド
- [ ] ルート結果画面: ルート選択セル、詳細ボタン
- [ ] スポット詳細画面: 電話ボタン、ウェブリンク、写真サムネイル

---

## 5. モーション・アニメーション

- [ ] `UIAccessibility.isReduceMotionEnabled` を確認し、アニメーションを簡略化
- [ ] 地図のアニメーション遷移を Reduce Motion 時にオフ
- [ ] ローディングスピナーは Reduce Motion 時に静的インジケーターに変更

---

## 6. その他の確認事項

### Switch Control

- [ ] 全画面で Switch Control による操作が可能
- [ ] フォーカス順序が論理的（左上から右下）

### Voice Control

- [ ] ボタン・リンクのラベルがVoice Controlで認識される
- [ ] 「タップ [ラベル名]」で操作できる

### キーボードナビゲーション（外部キーボード接続時）

- [ ] Tab キーでフォーカス移動が可能
- [ ] Enter/Space キーで選択操作が可能
- [ ] Escape キーでモーダル・シートを閉じられる

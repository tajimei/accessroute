## アプリの概要

AccessRoute は、車椅子・ベビーカー・杖歩行・高齢者など、さまざまな身体状況に合わせたバリアフリールートとアクセシブルスポットを提案する iOS アプリである。

### 主な特徴

- **パーソナライズされたルート検索**: ユーザーの移動手段や回避条件に基づき、最適なバリアフリールートを複数提案する
- **アクセシビリティスコア**: ルートやスポットごとに 0〜100 のスコアでバリアフリー度を可視化する
- **沿道スポット情報**: ルート沿いのトイレ、休憩所、エレベーターなどのバリアフリー施設を表示する
- **警告表示**: 階段、急な坂道、未舗装路など注意が必要な区間を事前に通知する
- **VoiceOver 対応**: 全画面で VoiceOver によるスクリーンリーダー操作に対応している

## 初回セットアップ（オンボーディング）

アプリを初めて起動すると、5ステップのオンボーディングが表示される。ここで登録した情報がルート検索やスポット提案に使用される。各ステップは「戻る」「次へ」ボタンで自由に行き来でき、最終ステップで「完了」ボタンを押すとプロファイルが保存される。

画面上部には進捗バーが表示され、現在のステップ番号と全ステップ数（例: 「ステップ 1 / 5」）を確認できる。

### ステップ 1: 移動手段の選択

自分の主な移動手段を1つ選択する。

| 選択肢 | 説明 |
|--------|------|
| 車椅子 | 車椅子で移動する方 |
| ベビーカー | ベビーカーを使用する方 |
| 杖 | 杖を使って歩行する方 |
| 徒歩 | 補助具なしで歩行する方 |
| その他 | 上記に該当しない移動手段の方 |

[画面イメージ: 2列グリッドで各移動手段がアイコン付きカードとして表示される。選択中のカードは青色でハイライトされる]

### ステップ 2: 同行者の選択

一緒に移動する同行者がいる場合、該当するものを全て選択する。該当がなければ何も選択せずに「次へ」を押してよい。

| 選択肢 | 説明 |
|--------|------|
| 子ども | 小さなお子さまを連れている |
| 高齢者 | 高齢の方と一緒に移動する |
| 障がい者 | 障がいのある方と一緒に移動する |

[画面イメージ: ステップ1と同様のカードグリッド形式。複数選択が可能]

### ステップ 3: 最大移動距離の設定

一度に移動できる最大距離をスライダーで設定する。100m から 5km の範囲で 100m 刻みで調整でき、画面中央に現在の設定値が大きく表示される。

[画面イメージ: 中央に距離のテキスト表示（例: 「1.5km」）、その下にスライダーが配置される]

### ステップ 4: 回避条件の選択

ルート上で避けたい条件を全て選択する。該当がなければ何も選択せずに進んでよい。

| 選択肢 | 説明 |
|--------|------|
| 階段 | 階段のあるルートを避ける |
| 急な坂道 | 急勾配の坂道を避ける |
| 混雑 | 混雑しやすいエリアを避ける |
| 暗い道 | 夜間に暗くなる道を避ける |

[画面イメージ: カードグリッド形式。複数選択が可能]

### ステップ 5: 希望条件の選択

ルート沿いにあると便利な施設の種類を全て選択する。該当がなければ何も選択せずに「完了」を押してよい。

| 選択肢 | 説明 |
|--------|------|
| トイレ | バリアフリートイレが近くにあるルートを優先 |
| 休憩所 | 休憩できるベンチや施設が近くにあるルートを優先 |
| 屋根あり | 屋根付きの経路を優先 |

[画面イメージ: カードグリッド形式。複数選択が可能。「完了」ボタンが青色で表示される]

最終ステップで「完了」ボタンを押すと、プロファイルがサーバーに保存され、ホーム画面に遷移する。

## ルート検索の使い方

### 目的地の入力方法

ホーム画面の上部に検索バーが表示されている。以下の方法で目的地を指定できる。

1. **テキスト検索**: 検索バーに施設名や住所を入力する。入力中は候補リストが表示されるので、目的の場所をタップして選択する
2. **地図上でタップ**: 地図上の任意の場所をタップして目的地に設定する

検索テキストをクリアしたい場合は、検索バー右端のクリアボタン（丸い×印）をタップする。

### ルート候補の見方

ルート検索を実行すると、条件に合致するルート候補が一覧で表示される。各ルートには以下の情報が含まれる。

| 項目 | 説明 |
|------|------|
| アクセシビリティスコア | 0〜100 の数値。数値が高いほどバリアフリー度が高い。プロファイルの設定（移動手段、回避条件等）に基づいて算出される |
| 距離 | ルート全体の総距離（メートル単位） |
| 所要時間 | ルート全体の推定所要時間（分単位） |
| 警告 | ルート上の注意点。例: 「この区間に階段あり」「急な坂道（勾配 8%）」など |

ルート検索時には、以下の3つの優先条件から1つを選択できる。

| 優先条件 | 説明 |
|---------|------|
| shortest（最短） | 距離が最も短いルートを優先する |
| safest（安全） | 安全性を重視したルートを優先する |
| accessible（バリアフリー） | アクセシビリティスコアが最も高いルートを優先する |

### ルート詳細の見方

ルート候補をタップすると、詳細画面に遷移する。詳細画面では以下の情報を確認できる。

- **ステップごとの案内**: 各区間の方向指示（例: 「北に 200m 直進」「右折して坂道を上る」）
- **区間情報**: 各ステップの距離、所要時間、路面の種類（舗装、砂利、土、不明）
- **バリアフリー情報**: 階段の有無、坂道の有無と勾配（%）
- **沿道スポット**: ルート沿いにあるバリアフリースポットの一覧

## スポット情報の見方

ルート検索結果や周辺スポット検索で表示されるスポットには、バリアフリーに関する詳細情報が付与されている。

### スポットカテゴリ

| カテゴリ | 説明 |
|---------|------|
| restroom | バリアフリートイレ |
| rest_area | 休憩所・ベンチ |
| restaurant | レストラン |
| cafe | カフェ |
| park | 公園 |
| kids_space | キッズスペース |
| nursing_room | 授乳室 |
| elevator | エレベーター |
| parking | 駐車場 |
| other | その他 |

### バリアフリーアイコンの意味

スポット詳細画面では、以下のバリアフリー情報がアイコンで表示される。

| アイコン | 項目 | 説明 |
|---------|------|------|
| 車椅子マーク | 車椅子対応 | 車椅子で利用可能な施設 |
| エレベーターマーク | エレベーターあり | エレベーターが設置されている |
| トイレマーク | バリアフリートイレ | 車椅子対応トイレが利用可能 |
| おむつ替えマーク | おむつ交換台 | おむつ交換台が設置されている |
| 授乳室マーク | 授乳室 | 授乳室が利用可能 |

### 床の種類

| 表示 | 説明 |
|------|------|
| flat | 平坦な床面 |
| steps | 段差あり |
| slope | 傾斜あり |
| mixed | 複合的（平坦な部分と段差・傾斜が混在） |

### アクセシビリティスコア

各スポットには 0〜100 のアクセシビリティスコアが付与されている。スコアが高いほど、バリアフリー対応が充実していることを示す。

## プロファイル編集

ホーム画面下部のプロファイルカードをタップすると、プロファイル設定画面に遷移する。ここではオンボーディングで登録した以下の項目を変更できる。

- 移動手段（車椅子、ベビーカー、杖、徒歩、その他）
- 同行者（子ども、高齢者、障がい者）
- 最大移動距離（100m〜5km）
- 回避条件（階段、急な坂道、混雑、暗い道）
- 希望条件（トイレ、休憩所、屋根あり）

変更内容は保存ボタンを押すとサーバーに反映され、以降のルート検索やスポット提案に即座に適用される。

## AIチャット機能

> この機能は Phase 2 で追加予定である。

AIチャット機能では、対話形式で移動のニーズを伝えることができる。テキストを入力すると、AIがトラベルコンシェルジュとして応答し、以下のサポートを行う。

- **ニーズの聞き取り**: 目的地、移動の目的、気になる点などを対話的に確認する
- **ルート提案**: 会話の内容からユーザーのニーズを自動抽出し、最適なルート検索を提案する
- **スポット紹介**: 条件に合ったバリアフリースポットを紹介する

AIが会話から抽出したニーズ（移動手段、回避条件など）は、そのままルート検索に反映される。AIの判断に基づき、追加の質問、ルート検索の実行、スポット表示のいずれかが自動的に提案される。
